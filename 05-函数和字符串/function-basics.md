# 函数基础详解

> 汇编中的函数调用机制
---

## 什么是函数？

### 高级语言中的函数

在 C 语言中，函数是这样的：

```c
int add(int a, int b) {
    return a + b;
}

// 调用
int result = add(3, 5);
```

**函数的三个要素：**
1. **参数** - 输入数据
2. **代码** - 执行逻辑
3. **返回值** - 输出结果

### 汇编中的函数

汇编没有内置的"函数"概念，函数本质上是：
1. **一段带标签的代码**
2. **使用 call/ret 进行跳转**
3. **通过栈或寄存器传递数据**

```nasm
; 定义函数
add_numbers:
    mov ax, [bp+4]    ; 读取参数 a
    add ax, [bp+6]    ; 加上参数 b
    ret               ; 返回

; 调用函数
push 3
push 5
call add_numbers
add sp, 4            ; 清理参数
```

## call 和 ret 指令

### IP 寄存器

**IP（Instruction Pointer）** 指令指针寄存器：
- 存储下一条要执行的指令地址
- 不能直接修改
- 通过 `jmp`、`call`、`ret` 间接修改

### call 指令

**功能：** 调用子程序

**执行步骤：**
```
1. push ip         ; 将返回地址压栈
2. jmp target      ; 跳转到目标地址
```

**图示：**
```
执行 call print_string 之前：
    IP → 0x7c30 (下一条指令地址)

执行 call print_string：
    1. IP 的值 (0x7c30) 被压入栈
    2. IP 被设置为 print_string 的地址

结果：
    IP → print_string 的地址
    栈顶: [0x7c30] (返回地址)
```

### ret 指令

**功能：** 从子程序返回

**执行步骤：**
```
1. pop ip          ; 从栈弹出返回地址
2. 跳转到返回地址
```

**图示：**
```
执行 ret 之前：
    IP → print_string 内某地址
    栈顶: [0x7c30] (返回地址)

执行 ret：
    1. 从栈弹出 0x7c30 到 IP
    2. 跳转到 0x7c30

结果：
    IP → 0x7c30 (返回 call 后的下一条指令)
```

## 栈与函数调用

### 为什么需要栈？

函数调用时需要保存：
1. **返回地址** - 函数执行完去哪里
2. **参数** - 传递给函数的数据
3. **局部变量** - 函数内部的临时数据
4. **保存的寄存器** - 需要恢复的寄存器值

栈提供了**自动管理**这些数据的机制。

### 调用过程示例

```nasm
; ===== 主程序 =====
mov si, hello      ; 准备参数
call print_string  ; 调用函数
mov al, 'X'        ; 返回后继续执行

; ===== 函数 =====
print_string:
    push si         ; 保存寄存器
.loop:
    ; ... 打印逻辑 ...
    jmp .loop
.done:
    pop si          ; 恢复寄存器
    ret             ; 返回
```

**栈的变化：**
```
调用前：
    SP → 0x8000

call 执行后：
    SP → 0x7FFE, [0x7FFE] = 0x7c30 (返回地址)

push si 执行后：
    SP → 0x7FFC, [0x7FFC] = si 的原始值

ret 执行前：
    SP → 0x7FFE, [0x7FFE] = 0x7c30

ret 执行后：
    SP → 0x8000, IP → 0x7c30
```

## 参数传递方式

### 方式 1：寄存器传递（快速但有限）

```nasm
; 调用
mov ax, 10
mov bx, 20
call add_func

; 函数
add_func:
    add ax, bx    ; ax = ax + bx
    ret
```

**优点：** 快速、简单
**缺点：** 寄存器数量有限，参数多时不适用

### 方式 2：栈传递（正规但复杂）

```nasm
; 调用（C 调用约定：参数从右到左压栈）
push 20          ; 第二个参数
push 10          ; 第一个参数
call add_func
add sp, 4        ; 清理参数（2 个参数 × 2 字节）

; 函数
add_func:
    push bp            ; 保存旧的基址指针
    mov bp, sp         ; 设置新的基址指针

    mov ax, [bp+4]     ; 读取第一个参数
    add ax, [bp+6]     ; 加上第二个参数

    pop bp             ; 恢复基址指针
    ret
```

**栈布局：**
```
高地址
    ┌──────────────┐
    │  参数 2 (20) │  [bp+6]
    ├──────────────┤
    │  参数 1 (10) │  [bp+4]
    ├──────────────┤
    │ 返回地址     │  [bp+2]
    ├──────────────┤
    │ 旧的 BP      │  [bp]
    └──────────────┘
低地址
    ↑
    SP
```

### 方式 3：混合方式（常用）

**少量参数用寄存器，大量参数用栈：**

```nasm
; 寄存器传递主要参数
mov si, string    ; 字符串地址
mov cx, length    ; 字符串长度

; 栈传递可选参数
push options
call string_func
add sp, 2
```

## 栈帧结构

### 什么是栈帧？

**栈帧（Stack Frame）** 是函数调用时在栈上分配的区域，包含：
- 参数
- 返回地址
- 保存的寄存器
- 局部变量

### 标准栈帧布局

```
高地址
    ┌──────────────────┐
    │  函数参数         │  正偏移 [bp+4, bp+6...]
    ├──────────────────┤
    │  返回地址 (IP)    │  [bp+2]
    ├──────────────────┤
    │  保存的 BP        │  [bp]
    ├──────────────────┤
    │  保存的寄存器     │  负偏移 [bp-2, bp-4...]
    ├──────────────────┤
    │  局部变量         │  [bp-6, bp-8...]
    └──────────────────┘
低地址
    ↑
    SP
```

### 函数序言和尾声

**序言（Prologue）** - 函数开始时的标准操作：
```nasm
push bp            ; 保存调用者的基址指针
mov bp, sp         ; 设置当前基址指针
sub sp, 4          ; 为局部变量分配空间（可选）
```

**尾声（Epilogue）** - 函数结束时的标准操作：
```nasm
mov sp, bp         ; 清除局部变量
pop bp             ; 恢复调用者的基址指针
ret                ; 返回
```

## 调用约定

### 什么是调用约定？

**调用约定（Calling Convention）** 规定：
1. 参数如何传递（寄存器/栈）
2. 参数顺序（从左到右/从右到左）
3. 谁清理栈（调用者/被调用者）
4. 哪些寄存器需要保存

### 常见调用约定

| 约定 | 参数传递 | 栈清理 | 寄存器保存 |
|------|----------|--------|-----------|
| **cdecl** | 栈（从右到左） | 调用者 | 调用者保存 |
| **stdcall** | 栈（从右到左） | 被调用者 | 调用者保存 |
| **fastcall** | 寄存器 + 栈 | 被调用者 | 调用者保存 |

### 简化版约定（引导程序用）

在我们的引导程序中，使用简化约定：
```nasm
; 参数：通过寄存器传递
; 返回值：ax 寄存器
; 保存寄存器：函数内必须保存 si, di, bp
```

## 函数调用示例

### 完整示例：带栈帧的函数

```nasm
; ===== 调用函数 =====
push 5          ; 参数 b
push 3          ; 参数 a
call add_numbers
add sp, 4       ; 清理参数
; 结果在 ax 中

; ===== 函数定义 =====
add_numbers:
    ; --- 序言 ---
    push bp            ; 保存旧 BP
    mov bp, sp         ; 设置新 BP

    ; --- 函数体 ---
    mov ax, [bp+4]     ; 读取参数 a
    add ax, [bp+6]     ; 加上参数 b

    ; --- 尾声 ---
    pop bp             ; 恢复旧 BP
    ret                ; 返回
```

### 无栈帧的简化函数

```nasm
; ===== 调用 =====
mov ax, 3
mov bx, 5
call add_simple
; ax 中是结果

; ===== 函数 =====
add_simple:
    add ax, bx    ; 直接使用寄存器
    ret           ; 简单返回
```

## 常见问题

### Q: 为什么需要保存和恢复寄存器？
**A:** 函数可能会修改寄存器的值，调用者可能需要这些值，所以需要保存。

### Q: call 和 jmp 的本质区别是什么？
**A:** call 会保存返回地址（压栈），jmp 不会。所以 call 可以配合 ret 返回。

### Q: ret 弹出的地址是从哪里来的？
**A:** 从栈顶。这个地址是由之前的 call 指令压入的。

### Q: 可以不用 ret 返回吗？
**A:** 可以用 `jmp [sp-2]` 或其他方式，但不推荐。ret 是标准做法。

### Q: 为什么有些寄存器需要保存，有些不需要？
**A:** 这是约定。通常 ax、cx、dx 是"易失"的（可以随便改），si、di、bp 是"非易失"的（需要保存）。

## 总结

| 概念 | 作用 |
|------|------|
| **call** | 跳转到函数，保存返回地址 |
| **ret** | 从函数返回 |
| **栈** | 保存返回地址、参数、局部变量 |
| **栈帧** | 每次函数调用在栈上的区域 |
| **调用约定** | 参数传递和寄存器使用的规则 |
