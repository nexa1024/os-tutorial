# 内存偏移 - 用大白话讲明白

> 03-引导内存配套概念文档

## 什么是内存偏移？

**简单说：内存偏移就是"距离起点的位置"**

---

## 生活类比：停车场的车位

想象一个停车场：

```
车位编号
0  1  2  3  4  5  6  7  ...
🚗 🚙 🚕 🚌 🚗 🚙 🚕 🚗
```

如果我问："红色车在哪里？"
- 你可以说："在 3 号车位"
- 这个 "3" 就是**偏移量**（offset）
- 它表示"从停车场入口（0号）开始，数过去 3 个位置"

**内存也是一样的道理！**

---

## 计算机内存布局

计算机内存就像一个超级大的停车场，每个位置都有一个编号：

```
地址     内容
────────────────────
0x0000   ...
0x0001   ...
0x0002   ...
...
0x7C00   引导扇区从这里开始 ← 我们关注的重点
0x7C01   ...
0x7C02   ...
...
0x7C2D   'X' 字符在这里
...
```

---

## 引导扇区的偏移问题

### 问题：汇编器认为从 0 开始

当你写汇编代码时：

```nasm
[org 0x7c00]

the_secret:
    db "X"
```

汇编器会认为：
- `the_secret` 的偏移 = 0x2D（从代码开头数过去的位置）

### 但是：BIOS 把代码加载到 0x7C00

实际运行时：
- 代码开头 = 0x7C00
- `the_secret` 的实际地址 = 0x7C00 + 0x2D = **0x7C2D**

### 图示

```
汇编器视角（以为代码从 0 开始）：
地址     0x00  0x01  ...  0x2D
内容     [代码 ... ]       'X'
                    ↑
                  偏移 0x2D

实际内存视角（代码在 0x7C00）：
地址     0x7C00 0x7C01 ... 0x7C2D
内容     [代码 ... ]        'X'
                         ↑
                       地址 0x7C2D
```

---

## org 指令的作用

`[org 0x7c00]` 告诉汇编器：

> "嘿，这段代码实际会被加载到 0x7C00，算地址时要加上这个偏移"

### 有 org vs 无 org

```nasm
; 无 org
the_secret: db "X"
; 汇编器认为 the_secret = 0x2D
; 但实际在 0x7C2D
; 访问时需要：mov al, [0x2D + 0x7C00]

; 有 org
[org 0x7c00]
the_secret: db "X"
; 汇编器认为 the_secret = 0x7C2D
; 访问时直接：mov al, [the_secret]
```

---

## 偏移计算公式

```
实际地址 = 基地址 + 偏移量
```

| 场景 | 基地址 | 偏移量 | 实际地址 |
|------|--------|--------|----------|
| 引导扇区开头 | 0x7C00 | 0x00 | 0x7C00 |
| the_secret | 0x7C00 | 0x2D | 0x7C2D |
| 任意数据 | 0x7C00 | X | 0x7C00 + X |

---

## 为什么要关心这个？

### 问题 1：不加偏移会怎样？

```nasm
; 错误写法（无 org）
mov al, [the_secret]   ; 访问 0x2D，但数据在 0x7C2D
; 结果：读到垃圾数据
```

### 问题 2：重复加偏移会怎样？

```nasm
; 错误写法（有 org 但手动加偏移）
[org 0x7c00]
mov bx, the_secret     ; bx = 0x7C2D
add bx, 0x7c00         ; bx = 0xF82D（错！）
mov al, [bx]
; 结果：访问错误地址
```

### 正确写法

```nasm
; 推荐
[org 0x7c00]
mov al, [the_secret]   ; 简单、正确
```

---

## 常见偏移值

| 用途 | 偏移地址 |
|------|---------|
| 引导扇区 | 0x7C00 |
| 引导扇区结束 | 0x7E00 |
| 传统的内核加载位置 | 0x7E00 |
| BIOS 数据区 | 0x400 - 0x500 |
| 中断向量表 | 0x000 - 0x3FF |

---

## 总结

1. **偏移 = 距离起点的位置**
2. **引导扇区固定加载到 0x7C00**
3. **使用 `[org 0x7c00]` 让汇编器自动处理偏移**
4. **手动加偏移容易出错，推荐用 org**
