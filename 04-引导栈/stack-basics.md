# 栈基础 - 用大白话讲明白

> 04-引导栈配套概念文档

## 什么是栈？

### 简单定义

**栈（Stack）**是一种"后进先出"（LIFO, Last In First Out）的数据结构。

---

## 生活类比：一摞盘子

### 最直观的理解

想象餐厅里的一摞盘子：

```
        ┌───┐  ← 栈顶（只能操作这里）
        │ 饭 │  最新放的
        ├───┤
        │ 菜 │
        ├───┤
        │ 汤 │
        ├───┤
        │ 盘 │  ← 栈底（最先放的）
        └───┘
```

### 操作规则

| 操作 | 术语 | 说明 |
|------|------|------|
| 放盘子 | **Push** | 只能放在最上面 |
| 拿盘子 | **Pop** | 只能拿最上面的 |
| 看顶部 | **Peek** | 看一眼最上面是什么，不拿走 |

**重要特点：**
- 不能从中间抽盘子
- 不能从最下面拿盘子
- 最后放的，最先拿走

---

## LIFO 原则

### LIFO = Last In First Out

```
操作序列：
1. Push A  → 栈: [A]
2. Push B  → 栈: [A, B]
3. Push C  → 栈: [A, B, C]
4. Pop     → 取出 C，栈: [A, B]
5. Pop     → 取出 B，栈: [A]
6. Pop     → 取出 A，栈: []
```

**输出顺序：** C, B, A（和输入顺序相反）

### 为什么需要 LIFO？

**场景：函数调用**

```
main() {
    A();      ← 调用 A
}

A() {
    B();      ← 调用 B
}

B() {
    return;   ← 返回到 A
}
// ← 返回到 main
```

使用 LIFO 栈保存返回地址：
1. 调用 A → 保存 "返回 main" 的地址
2. 调用 B → 保存 "返回 A" 的地址
3. B 返回 → 取出 "返回 A"（最新放的）
4. A 返回 → 取出 "返回 main"（之前放的）

完美匹配函数调用的后进先出特性！

## 计算机内存中的栈

### 栈的内存布局

x86 架构中，栈是内存中的一段区域：

```
内存地址（高）
    ↓
┌─────────────────────────┐
│     ... 其他 ...        │
├─────────────────────────┤ 0x8000 ← BP（栈底/基址）
│                         │
│      栈空间             │
│                         │
│     （向下生长）         │ ↓
│                         │
├─────────────────────────┤       ← SP（栈顶，动态变化）
│                         │
│     ... 其他 ...        │
└─────────────────────────┘
    ↑
内存地址（低）
```

**关键点：**
- BP（Base Pointer）：栈的基址，固定不变
- SP（Stack Pointer）：栈顶指针，随 push/pop 变化
- **栈向下生长**：SP 的值越来越小

## push 和 pop 详解

### push 操作

```nasm
push ax    ; 假设 AX = 0x1234
```

**CPU 做了什么：**
```
步骤 1: SP = SP - 2           ; 先向下移动 2 字节
步骤 2: [SP] = AX             ; 把 AX 的值写入 SP 指向的位置
```

**图示：**
```
push 前：
  SP → 0x8000
       ┌──────┐
       │  ... │
       └──────┘

push 0x1234 后：
  SP → 0x7FFE
       ┌──────┐
       │ 1234 │ ← 0x7FFE = 0x1234
       │  ... │
       └──────┘
```

### pop 操作

```nasm
pop bx     ; 弹出到 BX
```

**CPU 做了什么：**
```
步骤 1: BX = [SP]             ; 读取 SP 指向的值到 BX
步骤 2: SP = SP + 2           ; 向上移动 2 字节
```

**图示：**
```
pop 前：
  SP → 0x7FFE
       ┌──────┐
       │ 1234 │ ← 0x7FFE = 0x1234
       │  ... │
       └──────┘

pop 后：
  SP → 0x8000
       ┌──────┐
       │ 1234 │ ← 数据还在，但已失效（垃圾）
       │  ... │
       └──────┘
       BX = 0x1234
```

## 栈帧（Stack Frame）

### 什么是栈帧？

**栈帧**是函数调用时在栈上分配的一块区域，用于存储：
- 返回地址
- 函数参数
- 局部变量
- 保存的寄存器

### 栈帧结构

```
内存高地址
    ↓
┌────────────────────────────┐
│      上一个栈帧             │
├────────────────────────────┤
│      返回地址               │ ← 调用者压入
├────────────────────────────┤
│      保存的 BP              │ ← 被调用者压入
├────────────────────────────┤
│      局部变量               │
│      ...                    │
├────────────────────────────┤ ← BP 指向这里
│      参数 2                 │
│      参数 1                 │
├────────────────────────────┤
│      下一个栈帧             │
└────────────────────────────┘
```

### 函数调用过程

```c
int add(int a, int b) {
    int result = a + b;
    return result;
}

int main() {
    int sum = add(3, 5);
    return 0;
}
```

**栈的变化：**
```
1. 调用 add(3, 5) 前：
   push 5      ; 压入参数 b
   push 3      ; 压入参数 a

2. call add    ; 压入返回地址

3. 进入 add：
   push bp     ; 保存旧的 BP
   mov bp, sp  ; 设置新的 BP
   sub sp, 2   ; 为 result 分配空间

4. 返回时：
   mov sp, bp  ; 恢复 SP
   pop bp      ; 恢复 BP
   ret         ; 弹出返回地址，跳转回去

5. 回到 main：
   add sp, 4   ; 清除参数（5 和 3）
```

## 栈 vs 堆

### 两者的区别

| 特性 | 栈（Stack） | 堆（Heap） |
|------|------------|-----------|
| **生长方向** | 向下（高地址到低地址） | 向上（低地址到高地址） |
| **分配方式** | 自动（编译器） | 手动（malloc/free） |
| **分配速度** | 超快（只需移动 SP） | 较慢（需查找空闲块） |
| **大小限制** | 小（通常几 MB） | 大（可达 GB 级） |
| **生命周期** | 随函数调用自动创建/销毁 | 手动控制 |
| **用途** | 局部变量、函数调用 | 动态分配的大对象 |

### 内存布局图

```
内存高地址
    ↓
┌─────────────────────────┐
│      ... 其他 ...       │
├─────────────────────────┤
│                         │ ← 栈顶（SP）
│      栈（向下）         │
│                         │
├─────────────────────────┤
│      空闲空间           │
├─────────────────────────┤
│                         │
│      堆（向上）         │
│                         │
├─────────────────────────┤
│      ... 其他 ...       │
└─────────────────────────┘
    ↑
内存低地址
```

## 栈的应用场景

### 1. 函数调用

```nasm
call function    ; 压入返回地址，跳转到 function
function:
    ; ... 函数代码 ...
    ret          ; 弹出返回地址，跳转回去
```

### 2. 保存寄存器

```nasm
push ax          ; 保存 AX 的值
push bx          ; 保存 BX 的值

; ... 使用 AX 和 BX 做其他事 ...

pop bx           ; 恢复 BX
pop ax           ; 恢复 AX（注意顺序相反！）
```

### 3. 表达式求值

计算 `3 + 4 * 5`：

```
操作              栈状态
────────────────────────────
push 3           [3]
push 4           [3, 4]
push 5           [3, 4, 5]
mul              [3, 20]   ; 4 * 5 = 20
add              [23]      ; 3 + 20 = 23
```

### 4. 交换数据

```nasm
; 交换 AX 和 BX
push ax
mov ax, bx
pop bx
```

## 常见错误

### 错误 1：push/pop 不匹配

```nasm
push ax
push bx
pop ax    ; ❌ 顺序错了！AX 会得到 BX 的值
pop bx
```

**正确：**
```nasm
push ax
push bx
pop bx    ; ✅ BX 恢复
pop ax    ; ✅ AX 恢复
```

### 错误 2：栈溢出

```nasm
; 无限递归，没有终止条件
infinite_loop:
    push ax
    call infinite_loop    ; ❌ 栈会一直增长，最终溢出！
```

### 错误 3：忘记恢复栈

```nasm
function:
    push bp
    ; ... 函数代码 ...
    ; ❌ 忘记 pop bp
    ret
```

### 错误 4：直接访问栈位置错误

```nasm
push 'A'
push 'B'
mov al, [0x8000]    ; ❌ 0x8000 是栈底，不是数据！
                    ; 应该用 SP 或计算正确偏移
```

## 调试栈问题

### 如何检查栈的状态？

```nasm
; 打印 SP 的值（调试用）
mov ax, sp
; ... 打印 AX 的代码 ...
```

### 如何检测栈溢出？

在栈底设置魔数：
```nasm
mov bp, 0x8000
mov word [bp], 0xDEAD    ; 设置魔数

; ... 程序运行 ...

cmp word [bp], 0xDEAD     ; 检查魔数
jne stack_overflow        ; 如果不等于，说明溢出了
```

## 总结

### 核心概念

1. **LIFO**：后进先出
2. **Push**：压栈，SP 减小
3. **Pop**：出栈，SP 增大
4. **向下生长**：x86 栈从高地址向低地址生长

### 关键寄存器

| 寄存器 | 作用 |
|--------|------|
| SP | 栈顶指针，动态变化 |
| BP | 栈底指针，固定参考点 |

### 常见用途

- 函数调用和返回
- 保存/恢复寄存器
- 存储局部变量
- 表达式求值
