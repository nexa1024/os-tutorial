## 什么是栈？

### 栈的概念

**栈（Stack）**是一种后进先出（LIFO, Last In First Out）的数据结构。

想象一摞盘子：
```
    ┌───┐  ← 栈顶（最新放入的）
    │ C │
    ├───┤
    │ B │
    ├───┤
    │ A │  ← 栈底（最先放入的）
    └───┘
```

- 只能从顶部放入元素（push，压栈）
- 只能从顶部取出元素（pop，出栈）
- 最后放进去的最先拿出来

### 栈的用途

| 用途 | 说明 |
|------|------|
| **函数调用** | 保存返回地址、参数、局部变量 |
| **临时存储** | 保存寄存器值，方便后续恢复 |
| **表达式求值** | 计算复杂的数学表达式 |
| **递归** | 保存递归调用的上下文 |

## x86 栈寄存器

### 两个关键寄存器

| 寄存器 | 全称 | 作用 |
|--------|------|------|
| **SP** | Stack Pointer | 指向栈顶 |
| **BP** | Base Pointer | 指向栈底（基址） |

### 栈的生长方向

**重要：x86 的栈是向下生长的！**

```
内存地址（高）
    ↓
┌─────────────────┐
│   ... 其他 ...  │
├─────────────────┤ 0x8000 ← BP（栈底/基址）
│                 │
│   栈空间区域    │
│                 │
├─────────────────┤ 0x7FFE ← SP（栈顶，初始时）
│                 │
│   ... 其他 ...  │
└─────────────────┘
    ↑
内存地址（低）
```

当执行 `push` 时，SP 先减小，再写入数据（向下生长）

## push 和 pop 指令

### push - 压栈

```nasm
push ax
```

**执行步骤：**
1. SP = SP - 2（栈向下生长 2 字节）
2. 将 AX 的值写入 SP 指向的地址

### pop - 出栈

```nasm
pop bx
```

**执行步骤：**
1. 读取 SP 指向的地址的值，存入 BX
2. SP = SP + 2（栈向上收缩 2 字节）

## 图示理解栈操作

### 初始化栈

```nasm
mov bp, 0x8000    ; 设置栈底
mov sp, bp        ; 栈顶指向栈底（空栈）
```

```
内存地址        内容
────────────────────────────
0x8000    ← BP（栈底）
         ← SP（栈顶，指向 0x8000）
0x7FFE
```

### 执行 push 'A'

```nasm
push 'A'
```

```
执行步骤：
1. SP = SP - 2  → SP = 0x7FFE
2. 写入 'A' 到 [0x7FFE]

内存地址        内容
────────────────────────────
0x8000    ← BP
0x7FFE    ← SP, 0x7FFE = 'A'
```

### 执行 push 'B', push 'C'

```nasm
push 'B'
push 'C'
```

```
内存地址        内容
────────────────────────────
0x8000    ← BP
0x7FFE        'A'
0x7FFC        'B'
0x7FFA    ← SP, 0x7FFA = 'C'
```

### 执行 pop（出栈）

```nasm
pop bx    ; 弹出 'C' 到 bx
```

```
执行步骤：
1. 读取 [0x7FFA] = 'C' 到 bx
2. SP = SP + 2  → SP = 0x7FFC

内存地址        内容
────────────────────────────
0x8000    ← BP
0x7FFE        'A'
0x7FFC    ← SP, 0x7FFC = 'B'
0x7FFA        'C'（垃圾数据）
```

## 代码示例

### 示例：栈操作演示

**文件：** `boot_sect_stack.asm`

这个程序演示栈的基本操作，包括 push、pop 和直接访问栈内存：

```nasm
; 功能: 演示栈的基本操作
; 编译: nasm -f bin boot_sect_stack.asm -o boot_sect_stack.bin
; 运行: qemu-system-x86_64 boot_sect_stack.bin
; 预期输出: CBA（打印出栈的字符）

[org 0x7c00]        ; 告诉汇编器代码加载地址

; ===== 设置 BIOS 显示功能 =====
mov ah, 0x0e        ; BIOS int 0x10, ah=0x0e = 显示字符模式

; ============================================
; 初始化栈
; ============================================
; 注意：栈底应该设在远离 0x7c00 的地方
;       避免覆盖我们的引导扇区代码
mov bp, 0x8000      ; 设置栈底（基址）
mov sp, bp          ; 栈顶指向栈底（空栈状态）

; ============================================
; 压栈操作
; ============================================
push 'A'            ; 栈: [A]     (SP = 0x7FFE)
push 'B'            ; 栈: [A, B]  (SP = 0x7FFC)
push 'C'            ; 栈: [A, B, C] (SP = 0x7FFA)

; ============================================
; 直接访问栈内存（演示栈向下生长）
; ============================================
; 计算方式：
;   bp = 0x8000（栈底）
;   push 3 个 2 字节的值后，sp = 0x8000 - 6 = 0x7FFA
;   最后压入的 'C' 在 0x7FFA
;   倒数第二个 'B' 在 0x7FFC
;   第一个 'A' 在 0x7FFE

; 打印 0x7FFE 处的字符（第一个 'A'）
mov al, [0x7ffe]    ; 0x8000 - 2 = 0x7ffe
int 0x10            ; 打印 'A'

; 注意：不要尝试访问 [0x8000]
;       那是栈底，不是有效数据
mov al, [0x8000]    ; 未定义行为
int 0x10            ; 可能打印乱码

; ============================================
; 出栈操作（标准方式）
; ============================================
; pop 只能按字（2 字节）操作
; 所以需要一个辅助寄存器来处理低字节
pop bx              ; 弹出 'C' 到 bx
mov al, bl          ; 只取低字节
int 0x10            ; 打印 'C'

pop bx              ; 弹出 'B' 到 bx
mov al, bl
int 0x10            ; 打印 'B'

pop bx              ; 弹出 'A' 到 bx
mov al, bl
int 0x10            ; 打印 'A'

; ============================================
; 验证：已 pop 的数据变成垃圾
; ============================================
; 此时 SP 已经回到 0x8000
; 再次访问之前的栈位置会得到垃圾数据
mov al, [0x8000]
int 0x10            ; 打印垃圾数据

; ===== 程序结束 =====
jmp $               ; 无限循环

; ===== 引导扇区填充 =====
times 510-($-$$) db 0
dw 0xaa55
```

**编译与运行：**
```powershell
nasm -f bin boot_sect_stack.asm -o boot_sect_stack.bin
qemu-system-x86_64 boot_sect_stack.bin
```

**预期输出：** `A[CBA`（A 后面可能跟一个乱码，然后是 CBA）

## 栈内存布局详解

### 为什么栈底设在 0x8000？

```
内存布局：
┌────────────────────────────────┐
│ 0x0000                         │
│    ...                         │
├────────────────────────────────┤
│ 0x7C00  ← 引导扇区加载位置      │
│    ...                         │
│    引导扇区代码 (512字节)       │
│    ...                         │
├────────────────────────────────┤
│ 0x7E00  ← 引导扇区结束          │
│    ... 空闲空间 ...             │
├────────────────────────────────┤
│ 0x8000  ← 栈底（安全位置）      │
│    ↓                            │
│    栈向下生长                    │
└────────────────────────────────┘
```

**原因：**
- 引导扇区在 `0x7C00 - 0x7E00`（512 字节）
- 栈底设在 `0x8000`，避免覆盖引导代码
- 栈向下生长，不会与代码冲突

### push 操作的内存变化

```
每次 push 都会：
1. SP 减 2（向下移动 2 字节）
2. 写入 2 字节数据到 SP 指向的位置

push 'A':
  SP: 0x8000 → 0x7FFE
  [0x7FFE] = 'A'

push 'B':
  SP: 0x7FFE → 0x7FFC
  [0x7FFC] = 'B'
  [0x7FFE] = 'A'（保留）
```

## 常见问题

### Q: 为什么栈是向下生长的？
**A:** 这是 x86 架构的设计决定。历史原因是早期系统的内存布局安排。向上生长或向下生长都可以，只是设计选择不同。

### Q: push 和 pop 只能操作 16 位数据吗？
**A:** 在 16 位实模式下，push/pop 默认操作字（2 字节）。可以操作 32 位寄存器（如 eax）但在我们的引导环境中通常用 16 位。

### Q: 为什么 pop 后数据还在内存里？
**A:** pop 只是移动 SP 指针，不会清除内存。数据变成"垃圾"，下次 push 时会被覆盖。

### Q: BP 和 SP 有什么区别？
**A:**
- **SP（Stack Pointer）**：始终指向栈顶，随 push/pop 自动变化
- **BP（Base Pointer）**：指向栈的基址，通常用作参考点，不自动变化

### Q: 栈溢出是什么？
**A:** 当 push 太多数据，栈生长到覆盖其他内存区域（如代码区）时，就会发生栈溢出，导致程序崩溃。

### 新指令

| 指令 | 语法 | 说明 |
|------|------|------|
