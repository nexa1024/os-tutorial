# 中断机制详解

> 计算机系统中断机制理论知识

## 中断机制概述

### 什么是中断？

**中断（Interrupt）** 是一种计算机系统用于处理异步事件的机制。通过中断，外部设备或内部事件可以打断处理器当前的执行任务，转而执行相应的**中断服务程序（ISR）**，以快速响应和处理紧急事件。

### 中断 vs 轮询

| 对比项 | 中断机制 | 轮询机制 |
|--------|---------|---------|
| **响应方式** | 事件驱动，按需响应 | 主动检查，不断询问 |
| **CPU 效率** | 高，无需空转 | 低，浪费 CPU 周期 |
| **响应延迟** | 低，立即响应 | 高，取决于轮询间隔 |
| **实现复杂度** | 较高 | 简单 |

## 中断的类型

```
┌─────────────────────────────────────────┐
│             中断 (Interrupt)            │
├─────────────────────────────────────────┤
│  硬件中断               │  软件中断    │
│  ┌─────────────────────────────────┐   │
│  │ • 异步性                        │   │
│  │ • 外部设备触发（键盘、定时器）  │   │
│  │ • 优先级可设置                 │   │
│  └─────────────────────────────────┘   │
│                                         │
│  软件中断 (Software Interrupt)          │
│  ┌─────────────────────────────────┐   │
│  │ • 同步性                        │   │
│  │ • 程序显式触发（int 指令）      │   │
│  │ • 系统调用、异常处理           │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 硬件中断

**特点：**
- **异步性**：可以在任意时间发生
- **优先级**：可设定不同优先级
- **快速响应**：能迅速打断当前任务

**常见硬件中断源：**
- 键盘输入
- 鼠标移动
- 网络数据到达
- 定时器溢出
- 串口数据

### 软件中断

**特点：**
- **同步性**：由程序显式触发
- **可控性**：程序控制中断时机
- **功能多样**：系统调用、异常处理

**常见软件中断：**
- `int 0x10` - BIOS 视频服务
- `int 0x13` - BIOS 磁盘服务
- `int 0x16` - BIOS 键盘服务
- `int 0x21` - DOS 功能调用
- `int 0x80` - Linux 系统调用

## 中断工作原理

### 中断处理流程

```
┌──────────┐
│ 程序执行  │
└─────┬────┘
      │
      │ 中断发生
      ↓
┌─────────────────────┐
│ 1. 中断响应          │
│    暂停当前指令      │
│    保存上下文        │
│    (PC, 寄存器)     │
└─────────┬───────────┘
          ↓
┌─────────────────────┐
│ 2. 中断识别          │
│    查找中断向量表    │
│    获取 ISR 地址    │
└─────────┬───────────┘
          ↓
┌─────────────────────┐
│ 3. 执行 ISR         │
│    (处理中断事件)   │
└─────────┬───────────┘
          ↓
┌─────────────────────┐
│ 4. 中断返回          │
│    恢复上下文        │
│    继续原程序       │
└─────────┬───────────┘
          ↓
┌──────────┐
│ 继续执行  │
└──────────┘
```

## 中断服务例程（ISR）

### 什么是 ISR？

**中断服务例程（Interrupt Service Routine，ISR）** 是专门用于处理中断事件的函数。

### ISR 的特点

| 特点 | 说明 |
|------|------|
| **简短高效** | ISR 应尽量简短，避免复杂逻辑，减少中断延迟 |
| **重入性** | 支持嵌套中断时需要具备重入能力 |
| **上下文保存** | 需要保存/恢复被中断任务的寄存器 |
| **原子性** | 临界区操作不能被打断 |

### ISR 编程原则

```c
// 好的 ISR 示例
void timer_isr(void) {
    static volatile uint32_t counter = 0;

    // 1. 快速处理：只做必要操作
    counter++;

    // 2. 清除中断标志
    clear_interrupt_flag();

    // 3. 设置标志通知主程序
    data_ready = true;
}

// 不好的 ISR 示例
void bad_timer_isr(void) {
    // ❌ 在 ISR 中执行耗时操作
    printf("Timer interrupt!\n");  // 非常慢！

    // ❌ 复杂的计算逻辑
    for (int i = 0; i < 1000000; i++) {
        complex_calculation();
    }
}
```

## 中断向量表（IVT）

### 什么是 IVT？

**中断向量表（Interrupt Vector Table）** 是一个存储中断服务例程地址的表格。

### x86 实模式 IVT 结构

| 特性 | 值 |
|------|-----|
| **位置** | 内存地址 `0x0000` - `0x03FF` |
| **大小** | 1024 字节（256 个中断 × 4 字节） |
| **每个条目** | 4 字节（2字节段地址:2字节偏移地址） |

### IVT 内存布局

```
地址          内容
─────────────────────────────────
0x0000        中断 0 的 ISR 地址
0x0004        中断 1 的 ISR 地址
0x0008        中断 2 的 ISR 地址
...
0x03FC        中断 255 的 ISR 地址
```

### 常见 BIOS 中断向量

| 中断号 | 功能 | 用途 |
|--------|------|------|
| `0x00` | 除法错误 | CPU 异常 |
| `0x10` | 视频服务 | 显示字符、设置光标 |
| `0x11` | 设备列表 | 获取设备配置 |
| `0x12` | 内存大小 | 获取常规内存大小 |
| `0x13` | 磁盘服务 | 读写磁盘扇区 |
| `0x14` | 串口服务 | 串口通信 |
| `0x15` | 系统服务 | 扩展内存等 |
| `0x16` | 键盘服务 | 读取键盘输入 |
| `0x17` | 并口服务 | 打印机输出 |
| `0x18` | BASIC 入口 | ROM BASIC |
| `0x19` | 引导加载 | 重新引导系统 |
| `0x1A` | 时钟服务 | 读写系统时间 |
| `0x21` | DOS 功能 | DOS 系统调用 |

## 中断优先级与嵌套

### 中断优先级

不同的中断可以设定不同的优先级，决定响应顺序：

```
高优先级中断 ────┐
                  │ 可打断
中优先级中断 ────┤
                  │ 可打断
低优先级中断 ────┘
```

### 中断嵌套

**中断嵌套** 允许高优先级中断在处理低优先级中断时被响应。

```
主程序运行
    │
    │ 低优先级中断发生
    ↓
低优先级 ISR 执行
    │
    │ 高优先级中断发生
    ↓
高优先级 ISR 执行
    │
    │ 返回
    ↓
低优先级 ISR 继续执行
    │
    │ 返回
    ↓
主程序继续运行
```

### 优先级反转问题

**优先级反转**：低优先级任务持有高优先级任务需要的资源，导致高优先级任务被阻塞。

**解决方案：**
- 优先级继承（Priority Inheritance）
- 优先级天花板（Priority Ceiling）

## 中断相关概念

### 中断控制器（Interrupt Controller）

管理中断请求的硬件组件：
- **PIC（8259）**：可编程中断控制器
- **APIC**：高级可编程中断控制器

### 中断延迟（Interrupt Latency）

从中断发生到 ISR 开始执行的时间。

**优化方法：**
- 减少 ISR 执行时间
- 合理设置中断优先级
- 避免长时间禁用中断

### 中断屏蔽

在临界区内禁用中断：

```nasm
; 禁用中断
cli

; 临界区代码
...

; 重新启用中断
sti
```
